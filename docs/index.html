Quick Start

The fastest way to try Aster is the graph_test/graph_example.cc driver. This page follows the repository’s shortest build-and-run path, then shows how to move from the tiny example to larger random and power-law graphs.

Build

make static_lib

cd graph_test/ && make all

./graph_example --load_mode=tiny

The example driver lives in graph_test/graph_example.cc, and most helper workloads and tests live in graph_test/graph_benchmark_new.h.

Run the smallest end-to-end example

In code, --load_mode=tiny dispatches to TinyExample(). That routine adds vertices 1, 2, and 3, inserts edges 1 → 2 and 1 → 3, prints the outgoing edges of vertex 1, deletes edge 1 → 2, and prints the adjacency again.

./graph_example --load_mode=tiny

A successful run should show the neighbors of vertex 1, then show one fewer edge after the delete.

By default, --reinit=true, so the existing DB is destroyed before each run.

Try a larger random graph

random mode calls LoadRandomGraph(load_vertices, load_edges). If you also set --run_lookups=true, the driver follows the load with random adjacency lookups and prints average put latency and get latency.

./graph_example --load_mode=random --load_vertices=20000 --load_edges=200000 --run_lookups=true --lookup_count=100

--run_lookups=true tells the driver to run random neighbor lookups after loading the graph.
--lookup_count sets how many lookups to run. For example, --lookup_count=200 means the program will sample 200 vertices and read their adjacency lists.
By default, --run_lookups=false and --lookup_count=100, so the program only loads the graph unless you explicitly enable lookups.

Try a graph with hub nodes

powerlaw mode calls LoadPowerLawGraphNew(load_vertices, powerlaw_alpha). This is a good next step after the uniform random loader because it creates a graph where a few nodes have many more connections than others.

./graph_example --load_mode=powerlaw --load_vertices=20000 --powerlaw_alpha=2.0 --run_lookups=true --lookup_count=100

--powerlaw_alpha controls how unevenly connections are distributed across nodes.
Smaller values create a few highly connected hub nodes, while larger values make connections more evenly distributed.

Main knobs

Graph shape and workload

  • --load_mode=tiny|random|powerlaw
  • --load_vertices / --load_edges
  • --powerlaw_alpha
  • --run_lookups / --lookup_count

Storage and execution

  • --update_policy=0|1|2 → eager / lazy / adaptive
  • --encoding_type=0|1 → none / EFP
  • --is_directed=true|false
  • --reinit=true|false
  • --enable_bloom_filter / --direct_io

Defaults

Runtime defaults

  • --load_mode=tiny
  • --is_directed=true
  • --enable_bloom_filter=true
  • --direct_io=false
  • --reinit=true

Workload defaults

  • --run_lookups=false
  • --lookup_count=100
  • --load_vertices=20000
  • --load_edges=200000
  • --powerlaw_alpha=2.0
