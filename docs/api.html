Graph Operations

All graph operations go through the RocksGraph class defined in include/rocksdb/graph.h. This page is organized by operation type.

Setup

RocksGraph(Options& options,
           int edge_update_policy = EDGE_UPDATE_ADAPTIVE,
           int encoding_type = ENCODING_TYPE_NONE,
           bool auto_reinitialize = false,
           std::string db_path = "/tmp/demo")

Opens or creates a graph database at the given path.

Parameters

  • options — RocksDB Options. Set create_if_missing = true for first use.
  • edge_update_policy — how edge writes are handled. See constants.
  • encoding_type — adjacency list compression. ENCODING_TYPE_NONE or ENCODING_TYPE_EFP.
  • auto_reinitialize — if true, deletes any existing database at the path before opening.
  • db_path — filesystem path for the database directory.

Example

#include "rocksdb/graph.h"
#include "rocksdb/options.h"

rocksdb::Options options;
options.create_if_missing = true;

rocksdb::RocksGraph graph(
    options,
    EDGE_UPDATE_ADAPTIVE,
    ENCODING_TYPE_NONE,
    false,
    "/tmp/aster-demo"
);

Vertex Operations

Status AddVertex(node_id_t id)

Registers a new vertex. The vertex ID must be a positive 64-bit integer.

node_id_t CountVertex()

Returns the total number of vertices from graph metadata.

Example

rocksdb::node_id_t u = 1, v = 2, w = 3;

graph.AddVertex(u);
graph.AddVertex(v);
graph.AddVertex(w);

std::cout << graph.CountVertex();  // 3

Edge Operations

Status AddEdge(node_id_t from, node_id_t to)

Inserts a directed edge. The update method (eager, lazy, or adaptive) is determined by the edge_update_policy set at construction. 

Status AddEdgeLazy(node_id_t from, node_id_t to)

Forces a lazy (merge-operator) write regardless of the configured policy.

Status DeleteEdge(node_id_t from, node_id_t to)

Removes a directed edge between two vertices.

Status GetAllEdges(node_id_t src, Edges* edges)

Reads the full adjacency list for a vertex into an Edges struct. Call free_edges(edges) when done to release the allocated arrays. 

node_id_t CountEdge()

Returns the total number of edges from graph metadata.

All edge operations are directed. For undirected behavior, add both AddEdge(u, v) and AddEdge(v, u).

Example

graph.AddEdge(u, v);
graph.AddEdge(u, w);

rocksdb::Edges edges;
if (graph.GetAllEdges(u, &edges).ok()) {
  for (uint32_t i = 0; i < edges.num_edges_out; ++i)
    std::cout << edges.nxts_out[i].nxt << "\n";
  free_edges(&edges);
}

graph.DeleteEdge(u, v);

Batch Operations

Status AddEdgeBatch(const std::vector<std::pair<node_id_t, node_id_t>>& edges)

Inserts multiple directed edges in one call.

Status DeleteEdgeBatch(const std::vector<std::pair<node_id_t, node_id_t>>& edges)

Removes multiple directed edges in one call.

Status AddVertexWithEdges(node_id_t id,
                          std::vector<node_id_t>& out_neighbors,
                          std::vector<node_id_t>& in_neighbors)

Creates a new vertex with its full adjacency list in a single WriteBatch. Only writes this vertex's own adjacency — reverse edges on neighbors are not added automatically. 

std::pair<std::string, std::string> AddEdges(
    node_id_t from,
    std::vector<node_id_t>& tos,
    std::vector<node_id_t>& froms)

Builds one vertex record from full outgoing and incoming neighbor lists. Returns encoded key-value pair for direct insertion. 

void AddVertexForBulkLoad()

Increments the vertex count in graph metadata during bulk loading.

Properties

Vertices and edges can carry arbitrary key-value properties stored in separate column families (vprop_val, eprop_val).

Write

Status AddVertexProperty(node_id_t id, Property prop)
Status AddEdgeProperty(node_id_t from, node_id_t to, Property prop)

Read

Status GetVertexProperty(node_id_t id, std::vector<Property>& props)
Status GetEdgeProperty(node_id_t from, node_id_t to, std::vector<Property>& props)

Scan by property

std::vector<node_id_t> GetVerticesWithProperty(Property prop)
std::vector<std::pair<node_id_t, node_id_t>> GetEdgesWithProperty(Property prop)

Scan-based reverse lookup. Useful for testing and small workloads, not a secondary index.

Example

rocksdb::Property vp{.name = "kind", .value = "paper"};
graph.AddVertexProperty(u, vp);

std::vector<rocksdb::Property> vprops;
graph.GetVertexProperty(u, vprops);

rocksdb::Property ep{.name = "weight", .value = "1"};
graph.AddEdgeProperty(u, w, ep);

std::vector<rocksdb::Property> eprops;
graph.GetEdgeProperty(u, w, eprops);

auto matched = graph.GetVerticesWithProperty(vp);

Degree Queries

Exact

node_id_t GetOutDegree(node_id_t id)
node_id_t GetInDegree(node_id_t id)

Read the adjacency list and return the exact out-degree or in-degree.

Approximate

node_id_t GetDegreeApproximate(node_id_t id, int filter_type = 0)

O(1) degree estimation without reading the adjacency list. Uses the Morris Counter (FILTER_TYPE_MORRIS) by default.

The adaptive update policy uses these estimators internally to decide between eager and lazy writes.

Traversal

Status SimpleWalk(node_id_t start, float decay_factor = 0.20)

Random-walk traversal from a starting vertex. At each step the walk continues with probability 1 - decay_factor. Useful for graph sampling experiments.

Diagnostics

void GetRocksDBStats(std::string& stat)

Writes internal RocksDB statistics into a string for profiling and debugging.

void printLSM(int column = 0)

Prints the LSM-tree structure for the given column family to stdout.

DB* get_raw_db()

Returns a pointer to the underlying RocksDB instance for advanced use.

Types & Constants

Identifiers

using node_id_t = int64_t;   // vertex ID (positive)
using edge_id_t = int64_t;   // edge ID (internal)

Data structures

struct Edge {
  node_id_t nxt;              // neighbor vertex ID
};

struct Edges {
  uint32_t num_edges_out;     // outgoing count
  uint32_t num_edges_in;      // incoming count
  Edge* nxts_out;             // outgoing array
  Edge* nxts_in;              // incoming array
};

struct Property {
  std::string name;
  std::string value;
};

Edge update policies

EDGE_UPDATE_EAGER      0x0   // read-merge-write
EDGE_UPDATE_LAZY       0x1   // merge operator
EDGE_UPDATE_ADAPTIVE   0x2   // runtime decision (default)
EDGE_UPDATE_FULL_LAZY  0x3   // full lazy variant

Encoding types

ENCODING_TYPE_NONE     0x0   // raw adjacency lists
ENCODING_TYPE_EFP      0x1   // Elias-Fano compression

Degree filter types

FILTER_TYPE_NONE       0x0
FILTER_TYPE_MORRIS     0x2   // Morris Counter (default)
After calling GetAllEdges(), always call free_edges(&edges) to release the returned arrays. This frees in-process memory only — it does not delete data from the database.
