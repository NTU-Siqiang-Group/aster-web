API & Examples

Aster’s public graph interface is exposed in include/rocksdb/graph.h. The main abstraction is RocksGraph, which wraps RocksDB with graph-aware storage, update policies, adjacency encoding, degree estimation, and property support.

Where the API lives

Main entry points

  • include/rocksdb/graph.h — public graph API
  • db/graph.cc — implementation
  • graph_test/graph_benchmark_new.h — examples and validation paths

Storage context

  • Adjacency lists live in the default graph storage layer
  • Vertex properties live in vprop_val
  • Edge properties live in eprop_val
  • Graph metadata persists vertex and edge counts

Core types

Identifiers

  • node_id_t — 64-bit integer vertex identifier
  • edge_id_t — 64-bit integer edge identifier

Returned structures

  • Edges stores outgoing and incoming neighbors
  • Property stores a name and a value
The API is designed so the same storage engine can expose graph mutation, graph lookup, property storage, traversal, and diagnostic hooks from one interface.

Open a graph

A typical setup creates a RocksGraph instance with RocksDB options plus an update policy, an encoding type, and a database path. 

#include "rocksdb/graph.h"
#include "rocksdb/options.h"

rocksdb::Options options;
// Configure RocksDB options here if needed.

rocksdb::RocksGraph graph(
    options,
    EDGE_UPDATE_ADAPTIVE,
    ENCODING_TYPE_NONE,
    true,
    "/tmp/aster-demo"
);

Core graph API

Structure and mutation

  • AddVertex(node_id_t id)
  • AddEdge(node_id_t from, node_id_t to)
  • DeleteEdge(node_id_t from, node_id_t to)
  • AddEdges(...)
  • AddVertexForBulkLoad()

Read and count

  • GetAllEdges(node_id_t src, Edges* edges)
  • CountVertex()
  • CountEdge()
  • GetOutDegree(node_id_t id)
  • GetInDegree(node_id_t id)

Minimal graph example

This mirrors the same basic flow used by the tiny example in graph_test: add vertices, add edges, read neighbors, then delete one edge. 

rocksdb::node_id_t u = 1, v = 2, w = 3;

graph.AddVertex(u);
graph.AddVertex(v);
graph.AddVertex(w);

graph.AddEdge(u, v);
graph.AddEdge(u, w);

rocksdb::Edges edges;
if (graph.GetAllEdges(u, &edges).ok()) {
  for (uint32_t i = 0; i < edges.num_edges_out; ++i) {
    std::cout << edges.nxts_out[i].nxt << "\n";
  }
  free_edges(&edges);
}

graph.DeleteEdge(u, v);

Properties API

Aster supports both vertex properties and edge properties, along with reverse lookup helpers for matching vertices or edges by property.

Write and read properties

  • AddVertexProperty(node_id_t id, Property prop)
  • AddEdgeProperty(node_id_t from, node_id_t to, Property prop)
  • GetVertexProperty(node_id_t id, std::vector<Property>& props)
  • GetEdgeProperty(node_id_t from, node_id_t to, std::vector<Property>& props)

Reverse lookup helpers

  • GetVerticesWithProperty(Property prop)
  • GetEdgesWithProperty(Property prop)

Property example

rocksdb::Property vp{.name = "kind", .value = "paper"};
graph.AddVertexProperty(u, vp);

std::vector<rocksdb::Property> vprops;
graph.GetVertexProperty(u, vprops);

rocksdb::Property ep{.name = "weight", .value = "1"};
graph.AddEdgeProperty(u, w, ep);

std::vector<rocksdb::Property> eprops;
graph.GetEdgeProperty(u, w, eprops);

auto matched = graph.GetVerticesWithProperty(vp);

Policies, encoding, and degree estimation

Update policy

  • 0 = EDGE_UPDATE_EAGER
  • 1 = EDGE_UPDATE_LAZY
  • 2 = EDGE_UPDATE_ADAPTIVE

Encoding

  • 0 = ENCODING_TYPE_NONE
  • 1 = ENCODING_TYPE_EFP

Degree estimation

  • GetOutDegree() / GetInDegree() — exact
  • GetDegreeApproximate() — approximate
  • FILTER_TYPE_CMS and FILTER_TYPE_MORRIS are supported
The same knobs exposed by graph_example map directly to the API-level policy and encoding choices, so CLI experiments and embedded usage stay aligned.

Traversal and diagnostics

Traversal

  • SimpleWalk(node_id_t start, float decay_factor)

Diagnostics

  • GetRocksDBStats(std::string& stat)
  • printLSM(int column)
